# 클라우드

## 개념
```bash
클라우드 컴퓨팅이랑 인터넷 기반의 컴퓨팅을 말합니다.
인터넷 상의 가상화된 서버에 프로그램을 두고 필요할때마다 컴퓨터나 스마트폰 등에 불러와 사용하는 서비스입니다.
```

## 장점

```bash
1. 서버를 직접 구매할 때 고려해야 할 전력, 위치, 확장성을 고민하지 않는다.

2. 서버 세팅 등을 신경쓰지 않고 서비스 운영에만 집중가능하다.

3. 서버부하에 따른 실시간 확장성을 지원받을 수 있다.
```

## 서비스 제공 형태
```bash
클라우드 컴퓨팅은 서비스 제공 형태에 따라 퍼블릭 클라우드, 프라이빗 클라우드, 하이브리드 클라우드로 구분됩니다.

1. 퍼블릭 클라우드
>> 특정 기업이나 사용자를 위한 서비스가 아닌 인터넷에 접속 가능한 모든 사용자를 위한 클라우드 서비스 모델입니다.

>>서비스 제공자가 하드웨어, 소프트웨어를 관리합니다.

>> 데이터나 기능, 서버 같은 자원은 각 서비스에서 사용자 별로 권한 관리가 되거나 격리 되어, 서비스 사용자 간에는 전혀 간섭이 없다는 장점이 있다.

2. 프라이빗 클라우드<Private Cloud,사설 클라우드, 폐쇄 클라우드

>>제한된 네트워크 상에서 특정 기업이나 특정 사용자만을 대상으로 하는 클라우드로 서비스의 자원과 데이터는 기업 내부에 저장됩니다.

3. 하이브리드 클라우드(Htbrid Cloud)
>> 하이브리드 클라우드는 퍼블릭 클라우드와 프라이빗 클라우드를 병행해 사용하는 방식으로 여겨져 왔으나, 최근에는 개념이 모호해진 경향이 있어 클라우드(가상서버)와 온프레미스(물리서버)를 결합한 형태를 말하기도 합니다.
```

## 서비스 유형
- 서비스 관리 주체와 수준에 따라 클라우드 서비스를 IaaS,PasS, Saas로 구분할 수 있다.

```bash
1. Iaas(Infrasture as a Service)
>>서비스로서의 인프라를 뜻하는 Iaas는 사용자가 관리할 수 있는 범위가 가장 넓은 클라우드 컴퓨팅 서비스입니다.

>> 인프라 수준의 클라우드 컴퓨팅을 제공해 사용자가 서버 OS부터 미들웨어, 런타임, 그리고 데이터와 어플리케이션 까지 직접 구성하고 관리할 수 있습니다.

>> 서비스 제공업체는 데이터 센터를 구축해 다수의 물리 서버를 가상화해 제공하며, 네트워크, 스토리지, 전력등 서버 운영에 필요한 모든 것을 CSP(Cloud Service Provider)가 책임지고 관리합니다.

>>ex) Ec2,가비아 G클라우드

2. Paas
>>Paas는 '서비스로서의 플랫폼' 이라는 의미를 담고 있습니다. Paas는 Iaas 형태의 가상화된 클라우드 위에 사용자가 원하는 서비스를 개발할 수 있도록 개발 환경을 미리 구축해, 이를 서비스 형태로 제공하는 것을 의미합니다.

>> ex) ReadHat의 OpenShift

3. Saas

>> 서비스로서의 소프트웨어를 의미하는 SaaS는 클라우드 인프라 위에 소프트웨어를 탑재해 제공하는 형태로 IT인프라 자원분만 아니라 소프트웨어 및 업데이트 , 버그 개선 등의 서비스를 업체가 도맡아 제공합니다.

```
